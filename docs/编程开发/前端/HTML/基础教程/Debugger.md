## 宽松的语法

- 浏览器通过 **解释** HTML 显示结果，而非 **编译** 。
- 浏览器解释HTML很 **宽松** ，即便HTML出现语法错误，也能显示结果(只不过结果不是我们想要的)。
- 在web创建的早期，人们要发布的 **内容本身** 比 **语法正确** 更重要；如果从一开始浏览器解释HTML就很严格，web可能没有现在这么受欢迎。
